    project 2 work session 1: abstract notes on computer programming

    using programming languages vs common language
    Ambiguity
    Common languages contain things like 'english' or spin or body language or context. Programming languages seek to remove all those ambiguous parts of communication. Our goal in using programming language is not to communicate ideas, but to command a machine to do exactly as it is told, without question and without comment.

    Verbosity
    Using programming languages presents different limits on the nature and amount of verbs for that language. I think all verbs are singular in programming languages although the same singular verb can be executed multiple times in a particular line of code using control structures.

    stage 2 lesson 3

    Procedural Thinking

    Functions or procedures, while initially adding to complexity, reduces coding repetition. In the future when I have a library of code and the familiarity with that library coding efficiency will increase. I will recall that I already have written a particular code block to compare which numbers are smallest and which are largest and just copy and paste my already written code into the current project and move on down the road.

    Breaking procedures down to their elementary blocks is useful for solving big problems. For example our code 'bigger' that only takes two parameters can be used to find the result of any number of parameters and still yield one result; the biggest of that list.

    Making functions is thinking about the logic and steps required to complete a task. Example, given a list of numbers find the greatest value in that list and return it.

    Using the list is calling that function to find the value from a given list.

    Return statements in functions are not required. Data can be manipulated and acted upon without a procedure returning a value. However, returning values from a procedure provides data that can be tested against and provides clarity in our coding with expected results.

    structured data, mutability and lists

    This is pretty damn cool!! Lists and lists of lists is a super cool concept. The ability to change a list element at any time presents amazing possibilities including constructing databases, web scraping info and delivering an entire web page by issuing the command print page[0]

    Mutability or the ability to mutate a list provides amazing power that is at the root level of the language. the three most basic commands in programming; add, change, delete on the elements of list makes my head spin with glee.

    However, there is cause for concern. With great power, comes great responsibility and that is illustrated in the difference between 'append' and +. A programmer needs to pay special attention to appending a list to a list. This will not add each individual item of the list, it will add 1 element, the list!
